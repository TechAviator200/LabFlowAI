# LabFlow AI — Deploy Guide

Free-tier hosting: **Vercel** (frontend) + **Render** (backend) + **Supabase** (database, optional).

---

## Prerequisites

- Node.js 18+ and Python 3.11+ installed locally
- A [GitHub](https://github.com) account
- A [Vercel](https://vercel.com) account (free Hobby plan)
- A [Render](https://render.com) account (free tier)
- Optionally: a [Supabase](https://supabase.com) account (free tier) for persistent storage

---

## Step 0 — Verify it works locally

```bash
# Backend (runs in demo mode, no credentials needed)
cd labflowai/backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
uvicorn app.main:app --reload --port 8000
# → http://localhost:8000/health should return {"status":"ok","demo_mode":true}

# Frontend (in a new terminal)
cd labflowai/frontend
npm install
cp .env.example .env.local
# NEXT_PUBLIC_API_URL=http://localhost:8000
npm run dev
# → http://localhost:3000
```

## Step 2 — Deploy Backend to Render

### 2a. Create the service

1. Go to [dashboard.render.com](https://dashboard.render.com) → **New** → **Web Service**
2. Connect your GitHub repo (`labflow-ai`)
3. Render will detect `render.yaml` automatically — it sets the build and start commands

   If it doesn't auto-detect, set manually:
   - **Root directory**: `labflowai/backend`
   - **Build command**: `pip install -r requirements.txt`
   - **Start command**: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
   - **Health check path**: `/health`

4. Choose **Free** plan → **Create Web Service**

### 2b. Note your Render URL

After the service deploys, Render assigns a URL:
```
https://labflow-ai-api.onrender.com
```
(exact URL shown in your Render dashboard — copy it)

### 2c. Set environment variables in Render

Go to your service → **Environment** tab → add these:

| Variable | Value | Notes |
|---|---|---|
| `APP_ENV` | `production` | Pre-set via render.yaml |
| `DEMO_MODE` | `true` | Pre-set via render.yaml |
| `USE_MOCK_DB` | `false` | Pre-set via render.yaml |
| `SUPABASE_STORAGE_BUCKET` | `labflow-uploads` | Pre-set via render.yaml |
| `LLM_PROVIDER` | `openai` | Pre-set via render.yaml |
| `LLM_BASE_URL` | `https://api.openai.com/v1` | Pre-set via render.yaml |
| `LLM_MODEL` | `gpt-4o` | Pre-set via render.yaml |
| `SECRET_KEY` | *(auto-generated by Render)* | Pre-set via render.yaml |
| `CORS_ORIGINS` | `https://your-app.vercel.app` | **Set this after Step 3** |
| `SUPABASE_URL` | `https://xxxx.supabase.co` | Optional — see Step 4 |
| `SUPABASE_SERVICE_KEY` | `eyJh...` | Optional — see Step 4 |
| `LLM_API_KEY` | `sk-...` | Optional — enables AI parsing |

> **Without** `SUPABASE_URL` and `LLM_API_KEY`, the app runs in demo mode with in-memory storage and the heuristic parser. This is fully functional for demos.

### 2d. Verify backend is live

```bash
curl https://labflow-ai-api.onrender.com/health
# {"status":"ok","env":"production","demo_mode":true}
```

> **Render free tier note**: services spin down after 15 min of inactivity. The first request after spindown takes 30–60 seconds. This is normal.

## Step 3 — Deploy Frontend to Vercel

### 3a. Import the project

1. Go to [vercel.com/new](https://vercel.com/new) → **Add New Project** → import `labflow-ai`
2. **Root directory**: `labflowai/frontend`  ← important
3. Framework will be detected as **Next.js** automatically

### 3b. Set environment variables in Vercel

Before clicking **Deploy**, go to **Environment Variables** and add:

| Variable | Value |
|---|---|
| `NEXT_PUBLIC_API_URL` | `https://labflow-ai-api.onrender.com` |

Replace the value with your **actual Render URL** from Step 2b.

> This must be set **before** the first deploy — it's embedded in the build.

4. Click **Deploy**

### 3c. Note your Vercel URL

After deploy, Vercel assigns:
```
https://labflow-ai.vercel.app
```
(exact URL in your Vercel dashboard — copy it)

---

## Step 4 — Wire CORS (critical)

Go back to **Render** → your service → **Environment** tab → update:

| Variable | Value |
|---|---|
| `CORS_ORIGINS` | `https://labflow-ai.vercel.app` |

Then click **Manual Deploy** → **Deploy latest commit** on Render to apply.

This allows your Vercel frontend to call the Render backend without CORS errors.

> If you have a custom domain on Vercel, add both URLs comma-separated:
> `https://labflow-ai.vercel.app,https://yourdomain.com`

## Step 5 — Database setup with Supabase (optional)

Skip this step if you just want the demo — the app works without Supabase.

### 5a. Create a Supabase project

1. Go to [supabase.com](https://supabase.com) → **New project**
2. Choose a name (e.g. `labflow-ai`), set a database password, pick a region

### 5b. Run the schema migration

1. In Supabase → **SQL Editor** → paste the contents of:
   ```
   labflowai/backend/migrations/001_initial_schema.sql
   ```
2. Click **Run**

### 5c. Create the storage bucket

1. In Supabase → **Storage** → **New bucket**
2. Name: `labflow-uploads`
3. Set to **Private** (the backend uses the service key to access it)

### 5d. Copy credentials to Render

In Supabase → **Settings** → **API**:

| Supabase value | Render env var |
|---|---|
| Project URL | `SUPABASE_URL` |
| `service_role` key (secret) | `SUPABASE_SERVICE_KEY` |

> Never put `SUPABASE_SERVICE_KEY` in frontend env vars or commit it to git.

After updating Render env vars → **Redeploy**.

---

## Step 6 — Add LLM key (optional, for AI parsing)

Without an LLM key, the app uses the heuristic parser (rule-based, ~55% confidence).
With an LLM key, it uses GPT-4o (or your chosen model) for full AI extraction.

In **Render** → **Environment** → add:

| Variable | Value |
|---|---|
| `LLM_API_KEY` | `sk-...` (your OpenAI key) |
| `LLM_MODEL` | `gpt-4o` |

Then redeploy.

```
# Fill in when you have the accounts:

SUPABASE_SERVICE_KEY=eyJh...
LLM_API_KEY=sk-...
```

### Vercel (frontend)

```
NEXT_PUBLIC_API_URL=https://labflow-ai-api.onrender.com
```

That's it. The frontend only needs the one env var.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Dashboard shows "Failed to load" | CORS blocked | Set `CORS_ORIGINS` on Render to match your Vercel URL exactly |
| Backend 502 / unreachable | Render service asleep (free tier) | Wait 30–60s for cold start, or upgrade to paid |
| Parse returns empty result | No LLM key | Expected — heuristic parser still works, confidence ~45–55% |
| Data resets on Render restart | No Supabase configured | Add Supabase credentials to Render env vars |
| Vercel build fails on env var | `NEXT_PUBLIC_API_URL` not set before build | Add env var in Vercel dashboard, then Redeploy |
| TypeScript build errors | Type mismatch | Run `npm run build` locally to debug |
